## Root Causes

| Factor | Details |
|--------|---------|
| Documentation figures have **no `src`** | `src/content/index.ts:477-496` — all 3 `DocumentationFigure` entries omit the `src` field, so `Figure.tsx` always renders the camera placeholder |
| File naming vs ID mismatch | Files are `participant-1.jpg` / `participant-2.jpg`; data model IDs are `participant-01` / `participant-02`. The `src` value must use the **file name**, not the ID |
| `setting` figure has no file | No `public/setting.jpg` — this entry stays without `src` |
| Aspect ratio mismatch | Images are **2048×1536 (4:3)**, but `EngagementPage.tsx:53` passes `aspect-[5/4]` — this crops sides with `object-fit: cover` |
| No `404.html` for GH Pages | `public/404.html` does not exist — the SPA recovery script in `index.html` is non-functional; deep links fail with GitHub Pages' default 404 |

### 3. Create `public/404.html` — GitHub Pages SPA redirect

Standard SPA fallback: when a request hits a non-root path on GH Pages, `404.html` redirects with `?/<original-path>` as query parameter. The `index.html` recovery script (already present, lines 17-30) detects this and restores the real URL via `history.replaceState`.

Contents:
```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8" />
  <title>ePortfolio</title>
  <script>
    (function () {
      var segment = window.location.pathname.replace(/\/404\.html$/, '');
      var rest = segment.replace(/^\/[^\/]+/, '') || '/';
      var search = '?/' + rest.replace(/&/g, '~and~') + window.location.search;
      window.location.replace(segment.replace(/\/[^\/]*$/, '/index.html') + search + window.location.hash);
    })();
  </script>
</head>
<body></body>
</html>
```

This redirect ensures that when someone visits `/ePortfolio/introduction` directly, GitHub Pages serves `404.html`, which redirects to `/ePortfolio/index.html?//introduction`, and the `index.html` script restores the URL to `/ePortfolio/introduction`.
