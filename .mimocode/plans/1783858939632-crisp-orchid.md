## Step 1: `index.html` — favicon + apple-touch-icon

Add these two `<link>` tags inside `<head>`, after the existing `<meta>` tags and before the Open Graph block:

```html
<link rel="icon" type="image/png" href="%BASE_URL%icon.png" />
<link rel="apple-touch-icon" href="%BASE_URL%icon.png" />
```

Vite replaces `%BASE_URL%` at build time with `/ePortfolio/`, producing `/ePortfolio/icon.png`.

No existing favicon declarations to remove (there are none).

**Skip**: web manifest (none exists, task says don't introduce PWA setup).

## Step 2: `Sidebar.tsx` — replace BL monogram with icon

Current code at line 37-39:
```tsx
<span className="flex h-9 w-9 shrink-0 items-center justify-center rounded-md border border-hairline bg-surface-1 font-serif text-sm text-ink">
  BL
</span>
```

Replace with:
```tsx
<span className="flex h-9 w-9 shrink-0 items-center justify-center overflow-hidden rounded-md border border-hairline bg-surface-1">
  <img
    src={`${import.meta.env.BASE_URL}icon.png`}
    alt=""
    className="h-full w-full object-contain"
    decoding="async"
  />
</span>
```

- `alt=""` because the icon is decorative beside the visible "Beyond Limitations" brand text
- Keeps the same `h-9 w-9` (36×36) container, `rounded-md`, `border`, `bg-surface-1`
- `overflow-hidden` prevents any overflow from the image
- `object-contain` preserves aspect ratio, no distortion
- `decoding="async"` for non-blocking decode
- No `loading="lazy"` — icon is visible immediately in the sidebar
